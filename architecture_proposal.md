# AI Productivity Tracking & Clockify Automation System
## Architecture & Technical System Design

This document details the blueprint for a production-quality, intelligent, desktop-integrated productivity time tracking and Clockify integration system.

---

## 1. System Architecture

The project consists of three core logical layers:

```mermaid
graph TD
    subgraph Client System (Local)
        Agent[Python Desktop Agent]
        Browser[Browser Tab/URL Monitoring]
        Agent -->|Pings Active State| LocalQueue[SQLite Local Queue]
    end

    subgraph Backend Server (Cloud/Server)
        API[Express + TS Gateway]
        Socket[Socket.io WebSockets]
        Classifier[AI & Rule-based Classifier]
        Merger[Session Consolidation Engine]
        
        API --> Classifier
        API --> Merger
    end

    subgraph Database Layer
        Mongo[(MongoDB)]
    end

    subgraph Web App (Browser Dashboard)
        UI[React + TS SPA]
    end

    LocalQueue -->|REST POST /api/tracking/ping| API
    API -->|Reads/Writes| Mongo
    Socket -->|Real-time Activity Stream| UI
    UI -->|REST Queries / Settings / Tasks| API
    API -->|Syncs Time Entries| Clockify[Clockify API]
```

### Architecture Key Details
1. **State Isolation:** The Python Agent operates autonomously in the background. It buffers tracking pings locally in a lightweight database (SQLite or memory queue) and transmits them to the Node.js backend. This handles offline scenarios gracefully.
2. **REST for Ingestion, WebSockets for Control/Live View:** 
   * **Inbound Ingestion:** The Python Agent uses a robust HTTPS POST request because batch/offline recovery is straightforward.
   * **Real-time Dashboards:** A Socket.io layer connects the Backend to the Web UI. When the backend receives an active window ping from the client, it broadcasts the payload instantly to the user's active browser dashboard.
3. **Dual-Model Tracking Strategy:**
   * **Raw Events (`RawActivity`):** Tracks continuous high-resolution 5вЂ“10s snapshots. Used to calculate precise app usage breakdowns, peak hours, and distraction heatmaps.
   * **Work Sessions (`FocusSession`):** Consolidated timeline blocks (e.g., "Coding in VS Code" for 45 minutes). These are generated by a state-machine that groups similar raw events, merging minor switches (like a 30s StackOverflow check) and ignoring noise. Only consolidated sessions sync with Clockify.

## 3. Database Schema Design (MongoDB)

We will use MongoDB (via Mongoose). The design focuses on performance and index optimization for raw tracking events and task entities.

### 1. `users` Collection
Stores identities, password hashes, and user-configurable thresholds.
```typescript
interface IUser {
  _id: Types.ObjectId;
  name: string;
  email: string;
  passwordHash: string;
  createdAt: Date;
}
```

### 2. `settings` Collection
One-to-one relationship with `users`. Separated to prevent loading Clockify API keys and preferences during standard auth checks.
```typescript
interface ISettings {
  userId: Types.ObjectId;      // Indexed: Unique
  theme: 'light' | 'dark';
  idleTimeoutMinutes: number;   // Configurable: default 5
  mergeThresholdMinutes: number; // Configurable session continuity merge: default 2
  clockify: {
    apiKey: string;
    workspaceId: string;
    projectIdMap: Record<string, string>; // Maps WorkTrack category -> Clockify project ID
    autoSync: boolean;
  };
  privacy: {
    ignoreUrls: string[];      // Regex patterns of URLs to ignore (e.g. banking, personal mail)
    ignoreAppNames: string[];  // Executable filenames to ignore
  }
}
```

### 3. `raw_activities` Collection
Stores raw window tracking inputs sent by the Python agent. High volume; indexed by `userId` and `timestamp`.
```typescript
interface IRawActivity {
  userId: Types.ObjectId;      // Compound Index with timestamp
  timestamp: Date;             // Date and time of ping
  appName: string;             // e.g. "chrome.exe", "code.exe"
  windowTitle: string;         // e.g. "leetcode.com - Two Sum - Google Chrome"
  browserUrl?: string;         // e.g. "https://leetcode.com/problems/two-sum"
  isIdle: boolean;
  durationSeconds: number;     // Usually equal to tracker ping interval (e.g. 5)
  category: string;            // Resolved category (e.g., Coding, Entertainment)
}
```

### 4. `focus_sessions` Collection
Aggregated blocks of focus work.
```typescript
interface IFocusSession {
  userId: Types.ObjectId;      // Indexed
  startTime: Date;
  endTime: Date;
  durationSeconds: number;
  activityName: string;        // Principal activity (e.g., "Development on github.com")
  category: string;            // Dominant category
  syncedToClockify: boolean;
  clockifyTimeEntryId?: string;
}
```

### 5. `tasks` Collection
Task manager items with AI-suggested fields.
```typescript
interface ITask {
  userId: Types.ObjectId;      // Indexed
  title: string;
  description?: string;
  status: 'todo' | 'in_progress' | 'done';
  priority: 'low' | 'medium' | 'high';
  category: string;            // Associated category tag
  dueDate?: Date;
  aiSuggested: boolean;        // Flag if suggested by activity logs analytics
  createdAt: Date;
}
```

## 4. API Endpoints

All endpoints except registration and login require a valid JWT token passed in the `Authorization: Bearer <TOKEN>` header.

### Authentications (`/api/auth`)
*   `POST /register` - Body: `{ name, email, password }`. Creates a user + default settings.
*   `POST /login` - Body: `{ email, password }`. Returns JWT token and brief user profile.
*   `GET /me` - Returns details of the logged-in user.

### Settings (`/api/settings`)
*   `GET /` - Fetch user tracking preferences, ignore lists, and Clockify configuration.
*   `PUT /` - Update settings.

### Ingestion (`/api/tracking`)
*   `POST /ping` - Python Agent payload: `{ appName, windowTitle, browserUrl, isIdle, timestamp }`. Classifies, logs, merges, and triggers real-time WebSocket broadcasts.

### Sessions & Analytics (`/api/analytics`)
*   `GET /sessions/today` - Returns today's aggregated work timeline sessions.
*   `GET /dashboard/summary` - Returns stats for cards (Focus score, productive hours, top apps, category distribution).
*   `GET /charts/trends` - Query params: `?period=day|week|month`. Returns structured history data for charts.

### Task Management (`/api/tasks`)
*   `GET /` - Fetch user tasks.
*   `POST /` - Create a task.
*   `PUT /:id` - Update status, priority, or content.
*   `DELETE /:id` - Delete a task.
*   `GET /recommendations` - Analyze recent `raw_activities` and return proposed tasks (e.g., "Write Documentation for LeetCode progress").

### Clockify Synchronization (`/api/clockify`)
*   `GET /workspaces` - Fetches active Clockify workspace IDs using the saved API Key.
*   `POST /sync-session/:sessionId` - Force sync a specific `focus_session` to Clockify.

## 5. System Workflows & Logic

### 1. Smart Activity Classification Logic
*   **Layer 1 (Regex & Keyword Mapping):** Matches domains (e.g., `github.com` -> `Coding`), process names (e.g., `code.exe` -> `Coding`, `slack.exe` -> `Meetings`), and window titles (e.g., `debugging` -> `Debugging`).
*   **Layer 2 (NLP & Semantics):** For non-obvious titles (e.g., browser tab "Vite React App"), the backend processes the title text. In later phases, this triggers an embeddings-based semantic classifier or zero-shot categorization against standard work domains.

### 2. Session Consolidation & Merging Logic (State Machine)
Every 5 seconds, the server receives a tracking packet. Instead of writing separate sessions, it manages a running session:

```text
Incoming Ping (Category C, App A)
      в”‚

      в”‚     в”њв”Ђв–є NO: Create a new automatic session.
      в”‚     в””в”Ђв–є YES: 
      в”‚           в”њв”Ђв–є Same Category? 
      в”‚           в”‚     в””в”Ђв–є Extend session (duration += ping_duration, endTime = now).
      в”‚           в””в”Ђв–є Different Category?
      в”‚                 в”њв”Ђв–є Switch duration < Merge Threshold (e.g., 2 mins)?
      в”‚                 в”‚     в””в”Ђв–є Keep current session, add switch details to temp buffer.
      в”‚                 в””в”Ђв–є Switch duration >= Merge Threshold?
      в”‚                       в””в”Ђв–є Finalize/close current session, start new session of Category C.
```

## 6. Implementation Roadmap

### **Phase 1: Project Setup, Database, and Auth (Core Infrastructure)**
*   Initialize TypeScript project configuration for backend.
*   Setup MongoDB schemas and indexes.
*   Implement JWT sign/verify, register, login, and settings routes.
*   Initialize React SPA frontend using Vite + TS.

### **Phase 2: Premium Dashboard UI & Responsive Layout**
*   Create Sidebar layout, Dark/Light theme configuration, and responsive shell.
*   Construct mock pages for Analytics (using Recharts), Todos & Tasks, and Settings.
*   Enable Framer Motion glassmorphism styling and micro-animations.

### **Phase 3: Python Tracking Agent & Ingestion Gateway**
*   Build Python process monitor with `psutil` and active window listener with `pywin32`.
*   Implement keyboard and mouse listener via `pynput` for OS idle detection.
*   Write HTTP client with local caching (SQLite) to buffer and batch upload pings.
*   Connect Backend `/api/tracking/ping` endpoint to receive pings.

### **Phase 4: Real-time Socket Synchronization & Basic UI Integration**
*   Implement Socket.io in the backend and frontend.
*   On packet ingestion, broadcast real-time state to the UI to update the active dashboard clock and active app card instantly.

### **Phase 5: Classification Engine & Session Merger**
*   Build keyword-based classifier in backend services.
*   Build the session-merging state machine to synthesize raw pings into Focus Sessions.
*   Update frontend to display chronological focus timelines instead of raw logs.

### **Phase 6: Clockify Integration & Task Management**
*   Configure Clockify client handlers in the backend.
*   Build settings page inputs for Workspace selections and Project mappings.
*   Write Sync triggers to push finished sessions to Clockify.
*   Build Tasks page, CRUD APIs, and basic AI task recommendation algorithm.

### **Phase 7: Advanced Analytics & Optimization**
*   Finalize advanced charts, heatmaps, and summaries on the Analytics tab.
*   Optimize database query performances by applying Mongoose indexes.
*   Performance test Python agent to guarantee sub-1% CPU and Memory footprint.
